## TCP/IP通信の全体像

### 1. クライアント・サーバーモデル

```
┌─────────────────┐              ┌─────────────────┐
│   クライアント   │              │   サーバー      │

		│                                │
		│ ① 接続リクエスト                │
		│───────────────────────────────>│
		│                                │
		│ ② 接続確立                      │

		│                                │ ・大文字変換
```

## 2. ネットワーク層の構造

### TCP/IPの4層モデル

```
┌──────────────────────────────────┐
│  アプリケーション層               │  ← ClientErrorHandling
│  (あなたのプログラム)             │     ProtocolHandler
├──────────────────────────────────┤
│  トランスポート層                 │  ← TCP (信頼性保証)
│  (TCP/UDP)                       │     ポート番号: 11000
├──────────────────────────────────┤
│  インターネット層                 │  ← IP (アドレス管理)
│  (IP)                            │     IPアドレス: 192.168.1.100
├──────────────────────────────────┤
│  ネットワークインターフェース層    │  ← 物理的なネットワーク
│  (Ethernet, Wi-Fi)               │     (LANケーブル、無線)
└──────────────────────────────────┘
```

## 11. エラータイプの詳細

### ReceiveErrorType の種類と対処法

```
┌────────────────────────────────────────────────────────┐
│ None                                                   │
│ ・正常終了                                              │
│ ・対処: 不要                                            │
└────────────────────────────────────────────────────────┘

┌────────────────────────────────────────────────────────┐
│ DataTooLarge                                           │
│ ・1024バイトを超えるデータ                               │
│ ・原因: 送信データが大きすぎる                           │
│ ・対処: データを分割して送信                             │
└────────────────────────────────────────────────────────┘

┌────────────────────────────────────────────────────────┐
│ MissingEOF                                             │
│ ・<EOF>が見つからない                                   │
│ ・原因: 送信側がEOFを付け忘れた                          │
│ ・対処: ProtocolHandler.SendData()を使う                │
└────────────────────────────────────────────────────────┘

┌────────────────────────────────────────────────────────┐
│ ConnectionClosed                                       │
│ ・接続が切断された                                       │
│ ・原因: ネットワーク障害、サーバー停止                    │
│ ・対処: 再接続を試みる                                   │
└────────────────────────────────────────────────────────┘

┌────────────────────────────────────────────────────────┐
│ DataCorruption                                         │
│ ・データに異常がある                                     │
│ ・原因: 文字コード不一致、破損                           │
│ ・対処: 再送信を依頼                                     │
└────────────────────────────────────────────────────────┘
```

## 12. バイト数の計算例

### 英数字の場合

```
"Hello" = 5文字
H: 1バイト
e: 1バイト
l: 1バイト
l: 1バイト
o: 1バイト
─────────
合計: 5バイト
```

### 日本語の場合

```
"こんにちは" = 5文字
こ: 3バイト (UTF-8)
ん: 3バイト
に: 3バイト
ち: 3バイト
は: 3バイト
─────────
合計: 15バイト
```

### EOFを含む場合

```
"Hello World!Shimura<EOF>"

H e l l o 空 W o r l d ! S h i m u r a < E O F >
├─ 19文字 ─┤├─ 5文字 ─┤

英数字: 19バイト
<EOF>:  5バイト
─────────
合計: 24バイト ← 1024以下なのでOK
```

### 日本語を含む場合（注意が必要）

```
"日本語のテスト<EOF>"

日: 3バイト
本: 3バイト
語: 3バイト
の: 3バイト
テ: 3バイト
ス: 3バイト
ト: 3バイト
<EOF>: 5バイト
─────────
合計: 26バイト

※ 文字数は8文字だが、バイト数は26バイト
```

## 14. リソース管理の重要性

### 正しい終了処理

```
[リソースを確保]
	↓
Socket socket = new Socket(...);  ← メモリを使用開始
	↓
[使用]
socket.Connect(...);
socket.Send(...);
socket.Receive(...);
	↓
[解放]
socket.Shutdown(...);  ← 通信を停止
socket.Close();        ← メモリを解放
```

### 間違った例（リソースリーク）

```
Socket socket = new Socket(...);
socket.Connect(...);
socket.Send(...);
// ← ここで Close() を忘れる

結果: メモリが解放されない
	 → 何度も実行するとメモリ不足に
```

## 15. デバッグのヒント

### よくあるエラーと対処法

#### エラー1: "接続が拒否されました"

```
原因:
- サーバーが起動していない
- ポート番号が間違っている
- ファイアウォールでブロック

確認:
1. サーバー側を先に起動
2. ポート番号が一致しているか
3. Windows Defender を確認
```

#### エラー2: "タイムアウト"

```
原因:
- ネットワークが切断
- IPアドレスが間違っている
- サーバーが応答しない

確認:
1. ping コマンドで疎通確認
2. IPアドレスを再確認
3. ネットワーク接続状態
```

#### エラー3: "<EOF>が見つかりません"

```
原因:
- SendData() を使わず直接送信
- データが途中で切れた

確認:
1. ProtocolHandler.SendData() を使用
2. データ送信後に socket.Shutdown() していないか
```
