#### ErrorHandling (変更後)

```
┌──────────────────────────────────────────┐
│  HostErrorHandling.cs                    │
├──────────────────────────────────────────┤
│                                          │
│  // ProtocolHandlerで受信               │
│  var result =                            │
│      ProtocolHandler.ReceiveData(handler);│
│                                          │
│  // エラーチェック                       │
│  if (!result.Success)                    │
│  {                                       │
│      Console.WriteLine(                  │
│          $"エラー: {result.ErrorMessage}");│
│      // 接続を切断                       │
│      handler.Close();                    │
│      return;                             │
│  }                                       │
│                                          │
│  // 正常データを使用                     │
│  Console.WriteLine(                      │
│      $"受信: {result.Data}");            │
│                                          │
│  ✓ EOF検証                              │
│  ✓ サイズチェック                        │
│  ✓ データ異常検知                        │
└──────────────────────────────────────────┘
		 ↓
┌──────────────────────────────────────────┐
│  ProtocolHandler.cs (内部処理)           │
├──────────────────────────────────────────┤
│                                          │
│  public static ReceiveResult ReceiveData(│
│      Socket socket)                      │
│  {                                       │
│      var result = new ReceiveResult();   │
│                                          │
│      // 1. データ受信                    │
│      byte[] buffer = new byte[1024];     │
│      int bytesRec = socket.Receive(buffer);│
│                                          │
│      // 2. サイズチェック                 │
│      if (bytesRec > 1024)                │
│          return Error(DataTooLarge);     │
│                                          │
│      // 3. 文字列に変換                   │
│      string data = Encoding.UTF8         │
│          .GetString(buffer, 0, bytesRec);│
│                                          │
│      // 4. EOF検証                       │
│      int eofIndex = data.IndexOf("<EOF>");│
│      if (eofIndex == -1)                 │
│          return Error(MissingEOF);       │
│                                          │
│      // 5. データ抽出                     │
│      result.Data = data.Substring(0, eofIndex);│
│      result.Success = true;              │
│      return result;                      │
│  }                                       │
└──────────────────────────────────────────┘
```

## エラーハンドリングの追加

### エラーケースの比較

#### SimpleHost: エラー処理が弱い

```
┌─────────────────────────────────────────────┐
│  SimpleHost のエラー処理                     │
├─────────────────────────────────────────────┤
│                                             │
│  ケース1: サーバーが起動していない            │
│  ┌─────────────────────────────────────┐   │
│  │ try {                               │   │
│  │     socket.Connect(remoteEP);       │   │
│  │ }                                   │   │
│  │ catch (Exception e) {               │   │
│  │     Console.WriteLine(e.ToString());│   │
│  │     return;                         │   │
│  │ }                                   │   │
│  └─────────────────────────────────────┘   │
│  → ✓ 最低限のエラー処理はある                 │
│                                             │
│  ケース2: データが大きすぎる                  │
│  ┌─────────────────────────────────────┐   │
│  │ byte[] msg =                        │   │
│  │     Encoding.UTF8.GetBytes(st);     │   │
│  │ socket.Send(msg);                   │   │
│  └─────────────────────────────────────┘   │
│  → ✗ チェックなし (何バイトでも送れる)        │
│                                             │
│  ケース3: EOFがない                          │
│  ┌─────────────────────────────────────┐   │
│  │ int bytesRec = socket.Receive(bytes);│   │
│  │ string data =                       │   │
│  │     Encoding.UTF8.GetString(...);   │   │
│  └─────────────────────────────────────┘   │
│  → ✗ チェックなし (どんなデータも受け入れる)   │
│                                             │
│  ケース4: 接続が切断された                    │
│  ┌─────────────────────────────────────┐   │
│  │ socket.Receive(bytes);              │   │
│  └─────────────────────────────────────┘   │
│  → ✗ 検知できない (0バイトでも続行)           │
│                                             │
└─────────────────────────────────────────────┘
```

#### ErrorHandling: 詳細なエラー処理

```
┌─────────────────────────────────────────────┐
│  ErrorHandling のエラー処理                  │
├─────────────────────────────────────────────┤
│                                             │
│  ケース1: サーバーが起動していない            │
│  ┌─────────────────────────────────────┐   │
│  │ try {                               │   │
│  │     socket.Connect(remoteEP);       │   │
│  │ }                                   │   │
│  │ catch (Exception e) {               │   │
│  │     Console.WriteLine(              │   │
│  │         $"接続失敗: {e.Message}");   │   │
│  │     return;                         │   │
│  │ }                                   │   │
│  └─────────────────────────────────────┘   │
│  → ✓ わかりやすいエラーメッセージ             │
│                                             │
│  ケース2: データが大きすぎる                  │
│  ┌─────────────────────────────────────┐   │
│  │ if (!ProtocolHandler.SendData(...)) │   │
│  │ {                                   │   │
│  │     Console.WriteLine("送信失敗");   │   │
│  │     socket.Close();                 │   │
│  │     return;                         │   │
│  │ }                                   │   │
│  │                                     │   │
│  │ // ProtocolHandler内部で:          │   │
│  │ if (bytes.Length > 1024)            │   │
│  │     return false;                   │   │
│  └─────────────────────────────────────┘   │
│  → ✓ 1024バイト制限で事前チェック            │
│                                             │
│  ケース3: EOFがない                          │
│  ┌─────────────────────────────────────┐   │
│  │ var result =                        │   │
│  │     ProtocolHandler.ReceiveData(socket);│
│  │                                     │   │
│  │ if (!result.Success) {              │   │
│  │     if (result.ErrorType ==         │   │
│  │         ReceiveErrorType.MissingEOF)│   │
│  │     {                               │   │
│  │         Console.WriteLine(          │   │
│  │             "EOFが見つかりません");  │   │
│  │     }                               │   │
│  │ }                                   │   │
│  │                                     │   │
│  │ // ProtocolHandler内部で:          │   │
│  │ int eofIndex = data.IndexOf("<EOF>");│   │
│  │ if (eofIndex == -1)                 │   │
│  │     return Error(MissingEOF);       │   │
│  └─────────────────────────────────────┘   │
│  → ✓ EOF必須、なければエラー                 │
│                                             │
│  ケース4: 接続が切断された                    │
│  ┌─────────────────────────────────────┐   │
│  │ var result =                        │   │
│  │     ProtocolHandler.ReceiveData(socket);│
│  │                                     │   │
│  │ if (!result.Success) {              │   │
│  │     if (result.ErrorType ==         │   │
│  │         ReceiveErrorType.ConnectionClosed)│
│  │     {                               │   │
│  │         Console.WriteLine(          │   │
│  │             "接続が切断されました"); │   │
│  │     }                               │   │
│  │ }                                   │   │
│  │                                     │   │
│  │ // ProtocolHandler内部で:          │   │
│  │ if (bytesReceived == 0)             │   │
│  │     return Error(ConnectionClosed); │   │
│  └─────────────────────────────────────┘   │
│  → ✓ 0バイト受信を検知                      │
│                                             │
└─────────────────────────────────────────────┘
```

```

コードの重複 → 保守性が低い
```


## まとめ: 移行のメリット

### Before (SimpleHost)

```
❌ 問題点:
  - エラー処理が弱い
  - プロトコル規定なし
  - コードが重複
  - 保守が困難
  - データの完全性が保証されない

⚠️ リスク:
  - 異常データを受け入れる
  - メモリオーバーフロー
  - 予期しない動作
  - デバッグが困難
```

---

### After (ErrorHandling)

```
✅ 改善点:
  - 詳細なエラー処理
  - 統一されたプロトコル (<EOF>)
  - コードの再利用 (ProtocolHandler)
  - 保守が容易
  - データの完全性を保証

✨ メリット:
  - 異常データを検知
  - サイズ制限 (1024バイト)
  - 予測可能な動作
  - デバッグが容易
  - 拡張性が高い
```
